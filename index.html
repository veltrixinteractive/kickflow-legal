KickFlow Privacy Policy

Last Updated: March 15, 2026

This Privacy Policy explains how KickFlow and KickFlow Wear ("the Application") handle information when users install and use the application. Protecting user privacy is a core principle of the application's design. KickFlow is built to operate with minimal data usage and without collecting personal information.

1. No Personal Data Collection

KickFlow does not collect, store, transmit, process, or sell any personal information to external servers or third parties.

The Application does not collect or transmit:

  • Personal information
  • Email addresses or phone numbers
  • Location data
  • Device identifiers
  • Usage analytics
  • Advertising identifiers
  • Camera or microphone recordings

KickFlow does not require user accounts or registration and does not track users across devices or services.

2. Local Device Storage

Certain information required for application functionality is stored locally on the user's device. This includes:

  • User name and age (entered during onboarding)
  • Training history and workout progress
  • Custom workout programs
  • Experience points, level, and streak data
  • Unlocked badges and achievements
  • Body weight log entries
  • Language and theme preferences
  • Subscription status (Pro/Free)

All locally stored data:

  • Remains only on the user's device
  • Is never transmitted to external servers
  • Is not accessible to the developer
  • Is not shared with third parties

If the application is removed from the device, locally stored data may also be deleted.

3. Health Connect Integration

KickFlow may optionally integrate with Health Connect by Android to allow users to access fitness-related information.

If a user grants permission, the application may read the following data types through Health Connect:

  • Heart rate
  • Step count
  • Active calories burned

This data is handled under the following principles:

  • Access occurs only with explicit user permission
  • Data is used solely for displaying live health metrics and enhancing workout tracking within the application
  • Data remains on the user's device
  • Data is not transmitted to external servers
  • Data is not shared with the developer or third parties

Users may revoke Health Connect permissions at any time through their device settings.

4. Bluetooth Heart Rate Sensors

KickFlow may optionally connect to Bluetooth Low Energy (BLE) heart rate monitors (such as chest straps or smartwatch HR broadcast mode) to display real-time heart rate during workouts.

  • Bluetooth scanning occurs only with user-initiated action and appropriate system permissions
  • Heart rate data received via BLE is used only for live display and workout statistics within the application
  • No BLE data is stored beyond the current session or transmitted externally

5. Wear OS Companion App (KickFlow Wear)

KickFlow includes an optional companion application for Wear OS smartwatches (KickFlow Wear). The companion app communicates with the phone application using the Google Wearable Data Layer API.

Data exchanged between the phone and watch includes:

  • Heart rate readings
  • Step count
  • Calorie estimates
  • Punch and kick counts (detected via watch accelerometer and gyroscope)
  • Rep counts for bodyweight exercises
  • Current exercise name and workout elapsed time
  • Connection status

The Wear OS companion app accesses the following device sensors:

  • Heart rate sensor (TYPE_HEART_RATE)
  • Step counter (TYPE_STEP_COUNTER)
  • Linear accelerometer (TYPE_LINEAR_ACCELERATION)
  • Gyroscope (TYPE_GYROSCOPE)

All sensor data is processed locally on the watch for strike detection and workout tracking. Data is transmitted only between the paired phone and watch over the Wearable Data Layer (a direct device-to-device connection). No sensor data is transmitted to external servers, the developer, or third parties.

6. In-App Purchases

KickFlow offers an optional Pro subscription through Google Play Billing. Purchase transactions are processed entirely by Google Play.

  • KickFlow does not collect, store, or have access to payment information, credit card details, or billing addresses
  • Subscription status (active or inactive) is stored locally on the device
  • Purchase verification is handled directly between the application and Google Play services on the device

For information on how Google handles purchase data, refer to Google's Privacy Policy.

7. AI Features (Google Gemini)

KickFlow Pro includes AI-powered features such as KickFlow Bot (AI Coach) and AI Workout Generator. These features use the Google Gemini API to process user queries and generate responses.

When using AI features:

  • User messages and contextual fitness data (workout history, health metrics, weight log) are sent to Google's Gemini API servers for processing
  • Data is transmitted securely and used solely to generate AI responses within the app
  • Google may process this data according to their Gemini API Terms of Service and Google Privacy Policy
  • KickFlow does not store conversation data on external servers — chat history exists only in memory during the active session
  • AI-generated advice is for informational purposes only and should not be treated as professional medical, nutritional, or fitness advice
  • AI can make mistakes — always verify important information

Users who do not wish to use AI features can simply avoid using them. AI features require an active Pro subscription.

8. No Third-Party Data Sharing

KickFlow does not use third-party advertising networks, analytics SDKs, or tracking technologies that collect user data.

The Application does not share any information with advertisers, analytics providers, or external organizations. The only external data transmission occurs when using AI features as described in Section 7.

9. Data Security

Because KickFlow does not collect or transmit personal data to external servers, there is no external storage or processing of user information performed by the developer.

Users maintain full control of any data stored locally on their device. Phone-to-watch communication occurs over the secure Google Wearable Data Layer, which requires device pairing.

10. Age Restriction

KickFlow is a physical training and exercise application.

Due to the nature of fitness and physical activity, the Application is not intended for children under the age of 13.

Users under the age of 13 should not use this application unless supervised by a parent or legal guardian.
